# HubSpot プロパティ自動更新システム：要件定義書（バックエンド編）

## 1. システム概要
GCP（Google Cloud Platform）上のバッチ処理により、GA4およびGSCから顧客ごとのWebサイト指標を取得し、HubSpotの「取引（Deal）」レコード内の特定プロパティに自動的に書き込む。



---

## 2. 機能要件

### 2.1. レコード抽出機能
* **対象特定:** HubSpotから「特定のアナリティクスID（GA4計測ID等）」が入力されている取引レコードを抽出する。
* **フィルタリング:** 更新が必要なアクティブな案件（運用中の顧客）のみを対象とする。

### 2.2. 外部データ取得機能（Google連携）
* **GA4 Data API連携:** セッション数、総ユーザー数、エンゲージメント率、コンバージョン数などの指標を取得。
* **Search Console API連携:** クリック数、表示回数、平均掲載順位などの指標を取得。
* **期間指定:** 実行タイミングに応じて「前月分（1日〜末日）」または「直近7日間」などの期間を動的に指定する。

### 2.3. データマッピング・変換機能
* **単位変換:** パーセンテージ（%）や数値の型（String/Number）をHubSpotのプロパティ型に合わせて整形。
* **ID照合:** HubSpot上の「GA4プロパティID」をキーに、Google側の対象アカウントを正確に特定する。

### 2.4. HubSpot書き込み機能（CRM API）
* **一括更新（Batch Update）:** API制限を考慮し、複数のレコードをまとめて更新するリクエストを送信。
* **タイムスタンプ記録:** 「最終データ同期日」プロパティを更新し、いつのデータが反映されているか可視化する。

---

## 3. データ要件（マッピングリスト案）

スクリーンショットに基づいた、更新対象プロパティの定義例です。

| HubSpotプロパティ名（表示） | 内部名（API用例） | データソース | 取得指標（GA4/GSC） |
| :--- | :--- | :--- | :--- |
| **HPセッション** | `hp_sessions` | GA4 | `sessions` |
| **HP総ユーザー** | `hp_total_users` | GA4 | `totalUsers` |
| **HPエンゲージメント率** | `hp_engagement_rate` | GA4 | `engagementRate` |
| **HP CV** | `hp_cv` | GA4 | `conversions` (特定イベント) |
| **HP CVR** | `hp_cvr` | GA4 | `sessionConversionRate` |

> [!IMPORTANT]
> **管理用キープロパティ（必須）:**
> 各「取引」レコードに、`ga4_property_id`（GA4のプロパティID）と `gsc_site_url`（GSCのサイトURL）を保存する項目を事前に作成しておく必要があります。

---

## 4. 非機能要件

### 4.1. 実行スケジュール
* **頻度:** 毎月1日 AM 9:00（前月確定データの反映）。
* **任意実行:** 必要に応じてGCPコンソールから手動実行可能。

### 4.2. エラーハンドリング・監視
* **リトライロジック:** APIのレート制限（429 Error）が発生した場合、指数バックオフによる再試行を行う。
* **ログ:** Cloud Loggingに「成功した企業ID」「失敗した理由」を記録する。

### 4.3. セキュリティ
* **認証:** HubSpot非公開アプリのトークンおよびGoogle Cloudサービスアカウントの認証情報は、GCPの **Secret Manager** で安全に管理する。
